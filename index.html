FAQ

What is “phyX over Python”?
It reads like: “a small layer (phyX) built on top of Python.” Meaning: Python is the runtime and ecosystem; phyX is the opinionated system layer (rules, interfaces, workflow) that makes it behave like a product.
Do I need to be a programmer?
Yes—this is internal testing. The current target audience is technical users who can give precise feedback.
When will it be public?
When internal testing stabilizes the core workflow and documentation. No dates promised yet.
